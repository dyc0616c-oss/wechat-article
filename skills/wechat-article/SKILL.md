# WeChat Article Skill

按下面流程执行公众号写作任务。

## 触发规则（新增）

- 只要用户请求"写公众号/公众号改写/二创公众号/公众号排版"，默认自动启用本技能。
- 无需用户再次指定"用 wechat-article skill"。
- 除非用户明确要求改用其他技能或流程，否则始终按本技能执行。

## 存档规则（需确认）

- 每篇公众号交付后，询问用户是否需要存档
- 存档路径：`drafts/公众号合集/YYYY-MM-DD/`
- 存档格式：`.md` 与 `.docx` 双格式
- 文件名：`公众号：{标题}.md` 与 `公众号：{标题}.docx`
- 若当日文件夹不存在，先创建再保存
- 用户明确同意后才执行存档

## V2 增强规则（执行版）

1) 多源融合硬校验
- 出稿前必须检查：主链接 + 至少 2 条相关视角（支持/反对/中立可混合）
- 若未达到，需在回执中明确"当前为单源稿，建议补料后再发"

2) 通用排版闸门（全账号统一）
- 默认字数：800字左右（建议区间 700-900）
- 正文固定 4-5 大段
- 第一大段不加小标题；从第二大段开始每段必须有小标题
- 每大段 4-8 句话
- 每句话独立换行
- 每句话 15-40 字
- 默认行间距按 **3 倍** 处理
- HTML 排版版正文段落默认使用：`<p style="line-height:3;">...</p>`
- 重要词语/重要数据/总结句必须用 `！关键词` 形式标注（示例：`！稳定交付`、`！可复用SOP`）
- 配图位必须直接写在正文对应段落后（如：`🖼️【配图：...】`），不在文末单独列“配图建议”板块
- 最后一大段不加配图位

3) 标题摘要 AB 模块
- 每篇固定输出：悬念/利益/观点 各 1 条
- 标题 10-15 字，摘要 6-12 字
- 必须标注"推荐主用款"

4) 合规风险词扫描
- 出稿前扫描：绝对化承诺、攻击性措辞、无证据指控
- 命中后优先改写为证据型、条件型表达

5) 发布前清单（Checklist）
- 账号开头提示是否按 style 生效
- 账号文末引导是否按 style 生效
- 配图位是否完整
- 重点句是否标注
- CTA 是否存在
- 评分是否给出（含提分建议）

6) 风格切换档位
- 默认提供三档可切：稳健版 / 吃瓜版 / 观点版
- 用户未指定时默认稳健版

7) 传播优先模式（新增）
- 当用户明确表示“先传播、不过度学术化、案例可叙事化”时，切到传播优先模式：
  - 写作目标：传播效率 > 学术严谨
  - 证据策略：保留关键锚点，不堆砌出处细节
  - 叙事策略：冲突开场 + 反问推进 + 可执行结尾
  - 风险边界：不得编造事实；不做人身攻击；不作绝对化收益承诺
- 传播优先模式下仍需给“本版缺点/风险点”1-3条，避免失真。

8) 交付回执模板固定化
- 每次交付后固定回执：
  - 成稿摘要（1-2 句）
  - 评分（总分+关键分项）
  - 存档路径（md/docx）
  - 本次 skill 作用概括

## 账号配置（多账号支持，主控+风格卡）

本 skill 作为**主控 skill**，只负责流程编排：
- 读素材
- 选风格
- 生成标题/摘要/正文
- 输出可发布版本

账号风格不再内嵌在 SKILL.md，统一拆分到 `references/style-*.md`。

### 账号→风格映射（强制读取）

- 映射文件：`references/account-style-map.json`
- 用户未指定账号时，默认 `链小智ChainThink`
- 执行步骤：
  1. 解析用户指定的账号名
  2. 从 `account-style-map.json` 查出 style id
  3. 读取对应 `references/{style-id}.md`
  4. 按风格卡约束完成写作

### 当前支持账号（由映射文件维护）

- 链小智ChainThink
- 老韭要开喷
- Ai吹哨兵
- 迷因百科
- 账号E（预留）

### 使用方式

用户可以通过以下方式指定账号：
- “按公众号C发这篇”
- “这篇发老韭要开喷”
- “按链小智ChainThink风格改写”

如果用户只给风格名（如 `style-c`），按该风格卡直接执行。

### 老韭最终版对齐（新增）

当账号命中 `老韭要开喷` / `老韭`（即 `style-b`）时，额外强制执行：
- 首段无小标题，且前 3 句必须同时包含：冲突点 + 损失/荒诞感 + 直接判断
- 正文主轴固定为：`爆点 → 时间线 → 对冲 → 老韭结论 → 自保动作 → 互动收尾`
- 配图位直接内嵌正文段后，不再单独输出“配图建议”板块
- 语气允许犀利和吐槽，但结尾必须拉回理性判断，不得只喷不落地
- 至少给 2 个硬锚点（数字 / 时间 / 公告动作 / 监管动作）
- 每篇至少出现 1 次“你以为…其实…”或同等反转句

### 多账号锁定机制（新增，强制）

- 同一轮任务只允许绑定一个账号（`account_id`）和一个风格（`style_id`）。
- 在输出标题前，先回执：`已锁定账号：{account}｜风格：{style}`。
- 如用户中途切换账号，必须清空上一账号的开头/结尾/CTA缓存后再继续写。
- 若账号名未命中映射，先回退 default，并提示用户补充“账号名→style”映射。

### 五账号并行生产规范（新增）

当用户要求“同主题产出5个号版本”时，按以下顺序执行：
1. 先产出一份中性“母稿”（只保留事实与结构，不带账号口癖）。
2. 再按 `style-a`→`style-e` 依次改写，不得交叉复制开头和结尾。
3. 每个账号版本必须包含：
   - 标题3类（悬念/利益/观点）
   - 摘要1条（6-12字）
   - 正文1版（符合该style）
   - CTA与文末链接（按style）
4. 输出时按账号分块，块头固定：`## 账号：{name}（{style}）`。
5. 最后附“差异对照表”：语气差异 / 结构差异 / CTA差异（每项一句）。

## 0) 先读取参考文件（按需）

- 账号映射：`references/account-style-map.json`
- 账号风格卡：`references/style-a.md` ~ `references/style-e.md`
- 风格指纹：`references/style-fingerprint.md`
- 标题公式：`references/title-patterns.md`
- 开头结尾库：`references/opening-ending-library.md`
- 禁用词句：`references/banned-phrases.md`
- 五账号并行：`references/multi-account-workflow.md`

当用户说"按我的号风格写"时，先读账号映射 + 对应风格卡，再读取其余参考文件。

## 1) 先确认输入

向用户确认以下最小信息（缺失就补问，已给就直接开始）：

- 文章主题
- 目标读者（新手/从业者/泛用户）
- 目标（涨粉/转化/品牌表达/观点输出）
- 期望长度（默认约800字；中长文 1500-2500）
- 段落规模（默认 4-5 大段，除非用户指定）
- 是否需要"每句话后换行"版
- 排版标记方式（HTML样式 / emoji标注样式）
- 是否需要带数据、案例、金句、小标题风格

## 2) 产出顺序（固定）

默认按这个顺序输出，除非用户指定只要其中某一项：

1. 选题角度（1-3 个）
2. 标题候选（3-8 个）
3. 结构大纲（3-6 节）
4. 正文初稿
5. 公众号排版版（可直接粘贴）
6. 封面图 prompt

**如果已登录公众号后台并授权操作：**
7. 直接写入公众号编辑器并【保存为草稿】（默认动作），然后回执"已保存草稿，可刷新查看"。

> 不必先点"模板"。若弹出模板窗口，默认点"取消"，避免卡住。

## 3) 风格执行规则（账号驱动）

- 主 skill 不内置任何单账号专属文案。
- 用户指定“链小智 / 老韭 / ai / 迷因百科”等账号名时，必须先通过映射读取对应 style 文件后再写作。
- 若用户未指定账号，默认读取映射中的 default。
- 若 style 未配置某字段（如开头/文末/链接），使用中性默认值，不得回退到链小智专属内容。
- 同一篇稿件只允许使用一个账号 style，禁止混用。

## 4) 标题生成规则

每次至少给 3 类标题：

- 悬念型：留下信息缺口
- 利益型：明确读者收获
- 观点型：态度鲜明、可讨论

默认标题长度建议 10-15 字（用户未指定时优先执行）。
可选摘要长度建议 6-12 字（带钩子、强信息差）。
避免标题党，不夸大承诺。

## 5) 正文结构模板（默认）

1. 开场钩子（问题/冲突/反常识）
2. 场景故事（真实感，降低理解门槛）
3. 核心观点（1-3 条）
4. 例子或数据支撑（少量、关键）
5. 反方疑问与回应（增强可信度）
6. 结尾行动（给读者一个可执行动作）

## 6) 公众号排版规范（强制）

输出"可直接粘贴到公众号编辑器"的版本，必须遵守：

**注意：不再强制"先套模板"。** 仅在用户明确要求或稿件需要固定合规声明时才使用模板插入功能。

- 结构只允许：**小标题 + 正文**（禁止无序列表/有序列表）
- 开头段落不加小标题，只写背景引入
- 开头段落结束后插入第1张配图
- 从第2段开始使用小标题
- 每一段结束后插入1张配图（以配图位标注写在正文中）
- 最后一段结尾不加配图
- 不再单独输出“配图建议”独立板块
- 文末固定放微信群二维码（引导进群）
- 如用户要求"每句话后换行"，则正文按句分行（每句独立一行）

### 标题与重点样式（强制）

- 默认（HTML版）：
  - 正文段落：`<p style="line-height:3;">正文</p>`
  - 小标题：`<span style="color:#D0021B;"><strong><em>小标题</em></strong></span>`
  - 重点句：`<span style="color:#D0021B;"><strong><em>重点内容</em></strong></span>`
- 如用户要求"不要HTML"，切换 emoji 标注版：
  - 小标题前缀：`🔴✨【小标题（红色+加粗+斜体）】`
  - 重点句前缀：`🔴✨【重点句（红色+加粗+斜体）】`
  - 配图位前缀：`🖼️【配图X：...】`

### 开头与文末（账号驱动）
- 开头提示是否需要、具体文案内容：以对应 `style-*.md` 为准。
- 文末引导统一四行结构（全账号通用骨架）：
  1) 从什么什么，到什么什么，想什么什么？
  2) 群里更新最新内幕、拆产业逻辑，扫码进群吃瓜！
  3) 获取更多资讯新闻点击：
  4) 对应账号链接（由 style 指定）
- 若 style 未配置链接，第4行可留空或改为“关注账号获取后续更新”。

## 7) 改写任务规则

当用户提供原文并要求"二创/改写"时：

- 保留原意与事实，不改变核心结论
- 优先改结构和表达，不瞎加未经证实的信息
- 可新增：过渡句、类比、示例、行动建议
- 信息融合要求（默认）：
  - 不只基于单条链接改写
  - 至少补充 2-3 条相关观点（支持/反对/中立）后再提炼
  - 若无法抓取补充信息，需明确说明"仅基于单链接改写"
- 结构默认：
  - 4-5 大段
  - 第一段不加小标题，其余段有小标题
  - 每段 4-8 句
  - 句句换行，单句 15-40 字
  - 重要词语/数据/总结句用 `！重要词语！` 标注
- 最少提供两个版本：
  - A：稳健发布版
  - B：更有态度版

## 8) 数据与引用

- 需要数据时，优先使用用户给定数据
- 无可靠来源时，明确写"示例数据/估算"
- 不编造机构名、研究结论和具体百分比

## 9) 封面图 prompt 生成

当用户需要封面图时，输出一条中文 prompt：

格式：

`文章封面：{标题}。风格：简洁、现代、公众号头图、强对比、高可读中文标题排版、16:9、2K。主视觉：{核心意象}。颜色：{主色+强调色}。避免：复杂背景、过多文字、水印。`

若用户要命令行示例，再补充：

```bash
uv run .../generate_image.py --prompt "文章封面：[标题]" --filename cover.jpg --resolution 2K
```

## 10) 自动写入公众号时的稳定性规则（新增）

当任务要求"直接写入并保存草稿"时，执行必须遵守：

1. 先新建空白稿，不复用旧草稿正文。
2. 优先"本地组装完整正文 → 一次粘贴（paste）"写入；若一次粘贴失败，再退回 `type` 分段写入（每段 100-300 字）。
2.1 一次粘贴后立即 `snapshot` 校验字数是否明显增长；若无增长，判定为写入未生效。
2.2 退回分段 `type` 时，每段后立即 `snapshot` 校验一次字数递增。
3. 任何一步如果出现 `tab not found / ref失效 / timeout`：
   - 60 秒内仅允许一次重抓页面 ref 并重试；
   - 超过 60 秒或再次失败，立即停止自动化并回报错误点，不继续盲试。
4. 写完后必须执行"三重完整性校验"：
   - 开头锚点：匹配当前账号 style 的 opening（若该账号要求）
   - 结尾锚点：匹配当前账号 style 的 footer_link 或中性收尾
   - 结构阈值：满足 4-5 大段（首段无小标题，其余段有小标题）、每段4-8句、句句换行、单句15-40字、重点语句含 `！重要词语！` 标注
   任一不满足即判定失败。
5. 校验通过后才点"保存为草稿"；校验不通过不得声称完成。
6. 保存后回到"首页/近期草稿"再次核验：草稿摘要需能匹配新稿开头关键句；不匹配则判定可能被截断，必须回报失败。

## 11) 质量评分卡（100分制，新增）

每篇二创完成后，默认附一份评分（若用户要求"只要正文"可省略）：

- 选题价值（15）
- 信息与证据（20）
- 结构与节奏（15）
- 可读性与情绪张力（15）
- 实用价值（15）
- 公众号适配度（10）
- 合规与风险表达（10）

评分档位：
- 90-100：可直接发，爆款潜力高
- 80-89：可发，建议小修
- 70-79：结构可用，需重写关键段
- <70：不建议发，重做框架

## 12) 最终交付格式

默认按以下格式回复：

- 标题候选（10-15字）
- 摘要候选（6-12字）
- 大纲（4-5大段）
- 正文（普通版，按账号字数策略；正文内含配图位）
- 正文（公众号排版版：HTML或emoji标注；正文内含配图位）
- 封面图 prompt
- 质量评分卡（总分+提分建议）
- 可选：100 字摘要 + 3 条朋友圈短文案

始终优先可执行、可发布，避免空泛建议。
