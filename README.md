# WeChat Skills Repo

这是一个给 **公众号内容自动化** 用的仓库。

## 一句话总结

只要你提供：
- **5-10 篇用于学习风格的公众号素材**
- **微信开发者平台的 AppID / AppSecret / API IP 白名单**
- **基础账号配置**

这套东西就可以帮你实现：
- 按你的公众号风格自动写稿
- 自动把文章保存到公众号草稿箱

如果你想看更完整的详细教程，直接看仓库里的 Word 文档：
- **`WeChat-Skills-Usage.docx`**

如果你是第一次接触，不用先看一堆技术词，先记住一句话：

- `wechat-article`：**负责写稿**
- `wechat-draft`：**负责自动保存到公众号草稿箱**

也就是：

> 先让 AI 按你的公众号风格写文章，再自动存进公众号草稿箱。

---

# 这个仓库适合谁？

适合这些人：

- 想做公众号内容自动化
- 有多个公众号账号，想让每个号风格不一样
- 想把“写稿”和“存稿”拆开做
- 想把自己的公众号风格训练成可复用模板
- 想让 AI 写完后直接进入公众号草稿箱

如果你是纯小白，也能先照着这份 README 理解大概逻辑。

---

# 仓库里有哪两个 Skill？

## 1）wechat-article

这是“写稿 skill”。

它负责：
- 写公众号文章
- 二创改写
- 生成标题、摘要、正文
- 按不同账号风格生成内容
- 输出适合公众号排版的版本

它不负责：
- 不处理微信公众号开发者平台 API
- 不直接把文章写进公众号草稿箱

一句话理解：

**它解决“怎么写”。**

---

## 2）wechat-draft

这是“存稿 skill”。

它负责：
- 接微信公众号开发者平台 API
- 获取 / 刷新 `access_token`
- 上传封面图
- 自动保存文章到公众号草稿箱

它不负责：
- 不负责文章风格写作本身
- 不负责训练公众号文风

一句话理解：

**它解决“怎么存”。**

# 最常见的使用方式

如果你想实现：

**AI 写完文章 → 自动保存到公众号草稿箱**

那最常见的流程就是：

### 第一步：用 `wechat-article` 写稿
产出内容包括：
- 标题
- 摘要
- 正文
- 按账号风格生成后的版本

### 第二步：用 `wechat-draft` 存稿
把上一步写好的内容：
- 自动拿 token
- 自动上传封面图
- 自动调用微信 API
- 自动保存到公众号草稿箱

# 小白最关心：我要准备什么？

如果你只想先体验“写稿”，那你只需要：
- 下载 `wechat-article.skill`

如果你想真的实现“自动保存到公众号草稿箱”，那你还需要额外准备：

## `wechat-draft` 最少要 3 个东西

1. **AppID**
2. **AppSecret**
3. **API IP 白名单**

而且现在这几个东西，**统一按微信开发者平台的入口去获取和配置**。

也就是说：
- 不是只给 AppSecret 就够
- 必须是 **AppID + AppSecret + API IP 白名单** 三件套
- 缺一个都跑不起来

---

# 这些东西去哪里拿？

现在统一按 **微信开发者平台** 处理：

## 你需要从微信开发者平台拿到：
- AppID
- AppSecret
- API IP 白名单配置入口

## 你需要做的动作：
1. 登录微信开发者平台
2. 找到你的公众号应用
3. 获取 AppID
4. 获取 AppSecret
5. 把运行这套 skill 的机器出口 IP 加入 API IP 白名单

如果白名单没配好，最常见报错就是：
- `invalid ip not in whitelist`

所以如果你要跑真实样品，**优先先把白名单处理好**。

# style 是什么意思？

这个仓库里有一个很重要的设计：

> **一个公众号账号 = 一套风格规则（style）**

比如：
- 公众号A = 一种写法
- 公众号B = 一种写法
- 公众号C = 一种写法
- 公众号D = 一种写法

这样以后你不用每次都跟 AI 解释：

- 这个号该犀利一点
- 那个号该专业一点
- 另一个号该更像聊天

你只要提前把风格写好，后面直接说：

- “按公众号B发”
- “按公众号C风格改”
- “按我的公众号发”

系统就会自动读取对应的 style 文件。

---

# 映射表是干嘛的？

映射表就是“账号名 → 风格文件”的对应关系。

文件位置：
- `skills/wechat-article/references/account-style-map.json`

比如现在的逻辑大概是：
- `公众号A` → `style-a`
- `公众号B` → `style-b`
- `公众号C` → `style-c`
- `公众号D` → `style-d`

意思就是：

你说“按公众号B发”，系统就会去读 `style-b.md`。

# 怎么做自己的公众号风格？

这个是很多人最关心的部分。

最简单的方法不是“凭空想一个风格”，而是：

## 第一步：先准备你自己的样稿
建议找你过去写得最像自己的 **5-10 篇公众号文章**。

然后观察这几个问题：
- 你写什么选题最多？
- 开头喜欢先抛结论，还是先讲故事？
- 你更像聊天，还是更像分析？
- 标题偏悬念，还是偏观点？
- 你常用哪些口头禅？
- 结尾喜欢提问、总结，还是 CTA？

## 第二步：新建一个 style 文件
放到：
- `skills/wechat-article/references/`

比如你可以新建：
- `style-f.md`

## 第三步：把你的风格写成规则
建议至少写清楚：
- 账号定位
- 语气
- 开头规则
- 文章结构
- 标题规则
- 风险边界
- 结尾方式
- 示例段落

## 第四步：把账号绑定到映射表
修改：
- `skills/wechat-article/references/account-style-map.json`

比如：

```json
{
  "default": "style-a",
  "accounts": {
    "我的公众号": "style-f",
    "主号": "style-f",
    "测试号": "style-e"
  }
}
```

这样以后你直接说：
- “按我的公众号发”

系统就会自动去读 `style-f.md`。

# 常见注意事项

## 1. 只下载仓库，不代表立刻能自动存稿
因为 `wechat-draft` 仍然需要你自己配置：
- AppID
- AppSecret
- API IP 白名单
- `config.json`

## 2. 不要把真实密钥传到公开仓库
不要提交：
- 真实 `config.json`
- 真实 AppSecret
- 真实 access_token

仓库里只放：
- `SKILL.md`
- `config.example.json`
- `scripts/`
- `.skill` 打包文件

## 3. style 不是魔法，是“规则化你的习惯”
如果你的样稿本身风格不稳定，那 AI 也很难稳定复刻。

所以做 style 时，尽量选：
- 最像你自己
- 最稳定
- 最想长期复用
的文章来提炼。
